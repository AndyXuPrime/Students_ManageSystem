# 项目运行说明

本文以当前代码为准，整理本地启动、接口和数据结构，便于后续维护。

## 目录结构

- `gateway`：网关服务，端口 `8080`
- `User_service`：业务服务，端口 `8082`
- `Resource_service`：资源服务，端口 `8083`
- `UI`：前端 Vue 3 + Vite

## 本地依赖

- JDK 17
- MySQL 8.0+
- Redis
- Nacos 2.x
- MinIO
- Node.js 16+

## 启动顺序

1. 启动 Nacos、Redis、MySQL、MinIO
2. 启动 `gateway`
3. 启动 `User_service`
4. 启动 `Resource_service`
5. 进入 `UI` 执行 `npm install`，再执行 `npm run dev`

## 网关与前端

- 前端开发代理指向 `http://localhost:8080`
- 网关统一使用 `/api/**`
- 路由以当前 `gateway/src/main/resources/application.yml` 为准

## 已实现接口

### 登录

- `GET /api/auth/captcha`
- `POST /api/auth/login`

说明：当前登录只校验验证码，`username/password` 已接收但后端未参与鉴权。

### 业务数据

- `GET /api/student/page`
- `POST /api/student`
- `PUT /api/student`
- `DELETE /api/student/{sno}`
- `GET /api/class/list`
- `POST /api/class/add`
- `PUT /api/class/update`
- `DELETE /api/class/delete/{classno}`
- `GET /api/course/list`
- `POST /api/course/add`
- `PUT /api/course/update`
- `DELETE /api/course/delete/{cno}`
- `GET /api/teacher/list`
- `POST /api/teacher/add`
- `PUT /api/teacher/update`
- `DELETE /api/teacher/delete/{tno}`

### 资源管理

- `POST /api/file/upload?courseName=...`
- `GET /api/file/search?keyword=...`
- `DELETE /api/file/delete/{id}`

说明：资源上传后会写入 MinIO 和 MySQL `course_resource` 表；搜索按文件名或课程名模糊匹配。

## 当前实体表

- `student_table`
- `class_info`
- `course`
- `teacher`
- `course_resource`

## 当前代码约定

- `User_service` 的服务实现类放在 `com.javaee_hw.service.implement`
- `Teacher` 和 `ClassInfo` 目前保留显式 getter/setter，便于 IDE 直接识别
- `Resource_service` 仅保留 MinIO + JPA 的核心依赖，避免引入无用显式版本声明
